### Task 3
- `MintAgentCert` mirrors `auth_material.go::ensureClientCert` but with three deliberate differences: no on-disk persistence (returns PEM bytes), 24h lifetime instead of 1y (thumbprint pinning makes long-lived OK but tight ceiling caps blast radius), and CN equals the canonical agent name verbatim instead of `"clawker-cli"`.
- Added private `loadCAFrom(certPath, keyPath)` so MintAgentCert is testable without the consts-driven path resolution that `loadCA()` requires.
- Thumbprint is lowercase-hex SHA-256 over `cert.Raw` (the DER bytes), exactly what `crypto/sha256.Sum256(peerCert.Raw)` will produce on the CP side at Register. Tests assert that round-trip explicitly to lock the format down before it has a server-side reader.
- Tests reuse `EnsureAuthMaterial()` to provision the CA in an isolated `testenv.New(t)` instead of synthesizing a hand-rolled CA — cheaper and ensures MintAgentCert is exercised against the same material the CLI ships with.
- Adopted `AgentCert{CertPEM, KeyPEM, ThumbprintHex}` struct return per type-design review. Locks the co-derivation invariant at the type level so callers can't pair a thumbprint with a different cert.

### Task 4
- Plan suggested duplicating `RegisterCLIClient`'s 60 lines for `RegisterAgentClient`. Chose to extract a private `registerHydraClient(ctx, url, clientID, scope, jwkData, tlsCfg)` helper instead — the bodies differ in only two fields. Future Hydra clients land as one extra public function each.
- Both clients use the SAME JWK (the CLI's signing key — the CP container's bind-mounted public half). Distinct `client_id` + scope keeps the AuthZ surface clean even though the signing material is shared. This is a deliberate property: the CLI signs both `clawker-cli` and `clawker-agent` assertions with one key, but Hydra issues separate tokens with separate scopes.
- `cmd/clawker-cp/main.go` Step 5 now registers both clients sequentially. Both calls are idempotent on 409 so safe across CP restarts and ordering doesn't matter.
- Tightened the success path to 201 Created only — the previous CLI code accepted 200 OK too, which would have masked a misconfigured proxy returning 200 with an empty body as a registered-client success.

### Task 6
- The dockerevents informer publishes container deltas with `Lifecycle` strings. Initial draft hardcoded `"stopped"` in `agentregistry/subscribe.go` — refactored to expose `dockerevents.LifecycleStopped` (+ created/running/paused) so a future rename of the informer's vocabulary surfaces as a compile error in every consumer. Updated `containerLifecycleFromAction` and `containerLifecycleFromState` in dispatch.go to use the new constants.
- Eviction triggers: `DeltaRemoved` (Docker destroy/remove) AND `DeltaUpdated` where `Lifecycle == LifecycleStopped` (Docker die / stop / kill). Pause MUST NOT evict — the kernel hasn't torn down the socket and the existing mTLS connection is still valid. Codified by `TestSubscribe_DoesNotEvictOnPaused`.
- Subscribe uses informer's existing `Subscribe(filter)` API. No new event-bus surface — plan called this out explicitly.
- Subscribe consumer goroutine exits on EITHER ctx.Done OR informer-channel-close (cancel triggers the latter). The returned cleanup blocks on `<-done` so callers can deterministically wait, no goroutine leak.
- `Touch` on unknown thumbprint logs at debug rather than silently no-op'ing. The contract is that callers have already authenticated, so a miss usually means a race with eviction or a thumbprint-derivation bug — debug visibility helps catch the latter.
- Registry is keyed by `[sha256.Size]byte` (32-byte fixed array). Map equality on the key IS the identity check; no further matching is needed. String-keying would allocate + force hex-encoding; typed wrappers would lose `==` and map-key-ability.

### Task 5
- Constant-time PKCE compare alone is not enough — the original draft hashed the verifier AFTER the slot lookup, so an attacker probing for valid agent_names could distinguish "name unknown" from "name known, wrong verifier" by SHA-256 latency. Fixed by computing `pkceChallenge(verifier)` before any branching on `r.slots[agentName]`.
- Wrong-verifier path deliberately leaves the slot intact (TTL handles eviction). Plan calls this out — a buggy clawkerd transient failure should be able to retry; deletion-on-mismatch would let one bad call burn a legitimate registration.
- `Reserve` validates `agent_name != ""`, `ChallengeMethod == "S256"`, and `ExpiresAt > ReservedAt` so a buggy CLI can't reserve an unenforceable slot.
- `Stop` is `sync.Once`-guarded — drain-to-zero, test cleanup, and CP shutdown all converge here, so non-idempotent close-on-channel would panic.
- Sweep logs at info when it evicts >0 slots so an agent that announces but never registers shows up in operator logs (firewall break / container hang). Per-slot eviction stays at debug.
- `pkceChallenge` is package-private; reused by tests via same-package access. Earlier draft over-claimed it as "exposed at package level" — corrected.
- Default sweep period is `consts.AgentSlotTTL/2` so a slot is at most half its TTL past expiry before eviction. Earlier hardcoded 30s; now derived.

## Context for All Agents

### Background

Branch 4 introduces `clawkerd`, the per-container agent daemon, plus the auth + registration handshake between CLI, CP, and clawkerd. Branches 1–3 shipped the CP daemon, AdminService gRPC surface, Hydra-based OAuth2 for CLI calls, firewall ownership, and Docker events feeder/informer. Branch 4 wakes up the agent path that was scaffolded but never used.

**Threat model (load-bearing):** Agents inside managed containers are *coercible* — benign-intent processes that can be hijacked via prompt injection on untrusted content. Auth design defends against a coerced agent attempting to impersonate another agent or escalate scope. Identity is channel-bound (mTLS cert thumbprint stored at Register), not metadata-bound (request body alone).

**Trust roots:**
- CLI is the IdP. Generates and signs all per-agent material.
- CLI's CA at `<DataDir>/auth/ca/` already exists (signs CLI client cert + CP server cert today). Branch 4 extends to per-agent leaf certs.
- CLI's signing key at `<DataDir>/auth/cli/` already exists for `private_key_jwt` against Hydra. Branch 4 reuses the same key to sign assertions for the new `clawker-agent` Hydra client.
- CP mints nothing for agents.

**No backwards compat concerns:** clawkerd has never shipped. All POC code in `internal/clawkerd/protocol/v1/` and POC `Server.Register` (shared-secret auth) gets deleted in Branch 4.

### Identity binding chain

**Naming convention:** `agent_name` is the canonical full name `clawker.<project>.<agent>` (or `clawker.<agent>` for empty-project per existing convention). It is unique across the host. The wire protocol carries only `agent_name` — no separate `project` field. CLI is responsible for constructing the canonical name.

**Container_id is server-derived, never agent-asserted.** clawkerd does not pass `container_id` in any RPC body. CP derives it from the slot at Register and from the cert thumbprint registry entry on subsequent calls.

```
1. AnnounceAgent (CLI → CP, admin scope)
   - CLI mints per-agent mTLS cert+key (CN=<agent_name>, signed by CLI CA)
   - CLI computes SHA256(certDER) → expected_cert_thumbprint
   - CLI signs assertion JWT (iss=sub=clawker-agent, ES256, CLI signing key)
   - CLI gens 32-byte verifier, computes S256 challenge
   - CLI sends body to AnnounceAgent:
       {agent_name, container_id, expected_cert_thumbprint,
        code_challenge, code_challenge_method=S256}
   - CP stores slot keyed by agent_name, value carries
       {container_id, expected_cert_thumbprint, challenge, expires_at}, TTL 60s

2. CLI writes to tmpfs mount on container:
   - /run/clawker/bootstrap/cert.pem
   - /run/clawker/bootstrap/key.pem
   - /run/clawker/bootstrap/ca.pem (CP server cert trust)
   - /run/clawker/bootstrap/assertion.jwt
   - /run/clawker/bootstrap/verifier
   - All 0400 root:root
   Env into container: CLAWKER_AGENT_NAME, CLAWKER_CP_HYDRA_URL, CLAWKER_CP_AGENT_ADDR
   (NO CLAWKER_CONTAINER_ID, NO CLAWKER_PROJECT — clawkerd does not need them)

3. docker start

4. clawkerd (root, in entrypoint)
   - Reads tmpfs files
   - POST assertion to Hydra → access token (clawker-agent client, agent:self:register scope)
   - mTLS dial to CP AgentPort with cert+key, trust=ca.pem
   - AgentService.Register({agent_name, code_verifier}), bearer token attached
     (no container_id, no project, no version)
   - On success: idle on ctx until SIGTERM (no heartbeat, no polling)

5. CP AgentService.Register handler checks:
   (a) mTLS peer cert valid against CLI CA (handshake)
   (b) Bearer token introspect, scope = agent:self:register
   (c) Slot lookup by agent_name → slot exists, not expired
   (d) PKCE: S256(verifier) == slot.challenge (constant time)
   (e) SHA256(peerCert.Raw) == slot.expected_cert_thumbprint (constant time)
       — defense vs cert swap in tmpfs between announce + boot
   (f) docker.inspect(slot.container_id).Networks[clawker-net].IPAddress
       == peer.FromContext(ctx) IP
       — defense vs cert+verifier theft replayed from a different container
   (g) docker.inspect labels match slot's agent_name
       — defense vs label tampering after announce
   slot.Consume (atomic delete)
   registry.Add(thumbprint → entry{agent_name, container_id, registered_at, last_seen})
   Return ok

6. clawkerd deletes /run/clawker/bootstrap/{verifier,assertion.jwt}
   Keeps cert+key+ca on tmpfs (which dies with container) only — never on host disk

7. Liveness:
   - mTLS connection stays open while clawkerd is alive
   - CP detects clawkerd death via gRPC connection drop (TLS layer)
   - CP detects container death via dockerevents (B3 informer)
   - On Docker die/destroy → registry.EvictByContainerID
   - No heartbeat, no polling — both signals already reach CP
```

**Future RPCs (B5+):** when non-Register methods are added, the interceptor's identity-binding becomes: `thumbprint := SHA256(peerCert.Raw)` → `entry := registry.Lookup(thumbprint)` → ctx-bound entry. Handlers read from ctx, never trust request body. The registry data shape and Lookup API exist in B4 ready for that consumer; they're populated at Register and used by AdminService.ListAgents (Task 9b below) but otherwise idle until the first per-agent RPC lands.

**Why two checks at Register (e + f):** they defend independent attack vectors.
- (e) thumbprint-vs-slot defends against cert swap in tmpfs (CLI delivered cert X, attacker swapped to cert Y before clawkerd boot)
- (f) IP-vs-docker defends against cert+verifier theft replayed from a different container's network position

Together they pin both *which cert is valid* and *which container holds it*.

### Design patterns to follow

- **Factory noun pattern**: every Factory field returns a noun. CLI command code accesses via `f.AdminClient(ctx)` etc. Use this pattern for any new Factory-resolved deps.
- **mocks via moq**: every interface gets a `//go:generate moq` directive; mocks live under `<package>/mocks/`. Never hand-edit generated files.
- **Test isolation**: unit tests use `configmocks.NewBlankConfig()` / `configmocks.NewIsolatedTestConfig(t)`. Integration tests under `test/e2e/` use harness with `dev.clawker.test=true` labels for cleanup.
- **Logger**: `f.Logger()` in command code, `*logger.Logger` in constructors for library code. Never `logger.Fatal()` in Cobra hooks.
- **Path/consts**: never hardcode filenames or paths — use `cfg.X()` accessors and `consts.X` constants.
- **Storage schema**: when adding new config fields (e.g. `cp.AgentPort`), follow `internal/config/CLAUDE.md` + `.claude/rules/storage-schema.md` — struct tags with `yaml`, `label`, `desc`, `default`, optionally `required`.
- **Adversarial testing**: agents are coercible. Tests should include attack scenarios (replay, wrong thumbprint, expired slot, label mismatch). See `.serena/memories/adversarial-test-harness`.

### Rules

- Read `CLAUDE.md`, relevant `.claude/rules/*.md`, and the relevant package `CLAUDE.md` files before starting.
- Use Serena tools for code exploration. `initial_instructions` → `check_onboarding_performed` → `list_memories` → use `find_symbol` / `get_symbols_overview` / `search_for_pattern`. Read symbol bodies only when needed. Avoid reading entire files via `Read` unless necessary.

Author E2E tests, ensure they compile (`go vet ./test/e2e/...`), commit them, and let the user run them on the host. Same restriction for any test under `test/e2e/`, `test/whail/`, or anything that requires a real Docker daemon and not the moby mock transport.
- Follow existing test patterns in the package being modified.

- Apostles of TDD: tests before code where practical, but the project rejects strict TDD per the cp-initiative-status memo. Use integration tests + E2E tests + battle-tested mocks as the primary correctness story.

   ```go
   func (h *Handler) Register(ctx context.Context, req *agentv1.RegisterRequest) (*agentv1.RegisterResult, error) {
       // PRECONDITION: AuthInterceptor (Task 10) has already verified
       //   - mTLS peer cert valid against CLI CA
       //   - bearer token introspect + scope agent:self:register
       // and attached the peer cert + peer addr to context.

       peerCert, ok := controlplane.PeerCertFromContext(ctx)
       if !ok { return nil, status.Error(codes.Internal, "missing peer cert") }
       peerIP, ok := controlplane.PeerIPFromContext(ctx)
       if !ok { return nil, status.Error(codes.Internal, "missing peer addr") }
       thumbprint := sha256.Sum256(peerCert.Raw)

       // (a) PKCE consume — atomic.
       slot, err := h.slots.Consume(req.AgentName, req.CodeVerifier)
       if err != nil {
           return nil, status.Error(codes.PermissionDenied, "registration rejected")
       }

       // (b) Cert thumbprint match against what CLI announced.
       expected, err := hex.DecodeString(slot.ExpectedCertThumbprint)
       if err != nil || subtle.ConstantTimeCompare(thumbprint[:], expected) != 1 {
           h.log.Warn().Str("agent", req.AgentName).Msg("cert thumbprint mismatch at register")
           return nil, status.Error(codes.PermissionDenied, "registration rejected")
       }

       // (c) Docker IP cross-check — defense vs cert+verifier theft from
       //     a different container's network position.
       expectedIP, labels, err := h.dockerInspect(ctx, slot.ContainerID)
       if err != nil {
           h.log.Warn().Err(err).Str("container_id", slot.ContainerID).Msg("docker inspect failed at register")
           return nil, status.Error(codes.PermissionDenied, "registration rejected")
       }
       if !peerIP.Equal(expectedIP) {
           h.log.Warn().Str("agent", req.AgentName).Str("expected_ip", expectedIP.String()).Str("peer_ip", peerIP.String()).Msg("peer IP does not match container")
           return nil, status.Error(codes.PermissionDenied, "registration rejected")
       }

       // (d) Label cross-check — labels[dev.clawker.agent] must match.
       if labels[consts.LabelAgent] != req.AgentName {
           h.log.Warn().Str("agent", req.AgentName).Str("label", labels[consts.LabelAgent]).Msg("label mismatch at register")
           return nil, status.Error(codes.PermissionDenied, "registration rejected")
       }

       // Pin to registry.
       h.registry.Add(agentregistry.Entry{
           AgentName: req.AgentName,
           ContainerID: slot.ContainerID,
           CertThumbprint: thumbprint,
           RegisteredAt: time.Now(),
           LastSeen: time.Now(),
       })

       h.log.Info().Str("container_id", slot.ContainerID).Str("agent", req.AgentName).Msg("agent registered")
       return &agentv1.RegisterResult{}, nil
   }
   ```

   ```go
   func main() {
       log := logger.New(logger.Options{Filename: "clawkerd.log", LogsDir: "/var/log/clawker"})
       defer log.Close()

       if err := run(context.Background(), log); err != nil {
           log.Error().Err(err).Msg("clawkerd exited")
           os.Exit(1)
       }
   }

   func run(ctx context.Context, log *logger.Logger) error {
       // 1. Read bootstrap from tmpfs.
       boot, err := readBootstrap(consts.BootstrapDir)
       if err != nil { return fmt.Errorf("read bootstrap: %w", err) }

       // 2. Resolve env. Only three vars — clawkerd doesn't need
       // container_id (server derives) or project (encoded in agent_name).
       hydraURL := os.Getenv(consts.EnvClawkerdHydraURL)
       agentAddr := os.Getenv(consts.EnvClawkerdAgentAddr)
       agentName := os.Getenv(consts.EnvClawkerdAgentName)
       // (validate all non-empty; error otherwise)

       // 3. Build mTLS dial config.
       tlsCfg, err := buildTLSConfig(boot.CertPEM, boot.KeyPEM, boot.CACertPEM)
       if err != nil { return fmt.Errorf("tls: %w", err) }

       // 4. Exchange assertion for access token.
       token, ttl, err := auth.ExchangeAssertionForToken(ctx, hydraURL, boot.Assertion, agentScopes(), hydraTLSCfg)
       if err != nil { return fmt.Errorf("hydra exchange: %w", err) }

       // 5. Dial CP agent listener.
       conn, err := grpc.NewClient(agentAddr,
           grpc.WithTransportCredentials(credentials.NewTLS(tlsCfg)),
           grpc.WithUnaryInterceptor(bearerInterceptor(token)),
       )
       if err != nil { return fmt.Errorf("dial cp: %w", err) }
       defer conn.Close()

       agentClient := agentv1.NewAgentServiceClient(conn)

       // 6. Register. No container_id, no project, no version.
       _, err = agentClient.Register(ctx, &agentv1.RegisterRequest{
           AgentName: agentName,
           CodeVerifier: boot.Verifier,
       })
       if err != nil { return fmt.Errorf("register: %w", err) }

       // 7. Delete the verifier (single-use, spent). Assertion stays —
       //    the token source needs it for refresh on token expiry or
       //    auth failure on future RPCs. Cert+key+CA stay too (mTLS).
       //    All four live on tmpfs which dies with the container.
       _ = os.Remove(filepath.Join(consts.BootstrapDir, consts.BootstrapVerifierFile))

       // 8. Idle. Connection alive while we are. SIGTERM → ctx cancel → exit.
       //    No heartbeat — CP knows liveness via dockerevents + connection state.
       //    Future branches add a CP→clawkerd command stream here; for B4
       //    this is just a connection holder.
       log.Info().Str("agent", agentName).Msg("registered, idle")
       <-ctx.Done()
       return nil
   }
   ```

3. **Token refresh** — basic client hygiene, not deferred. clawkerd uses the shared `auth.TokenSource` (extracted in step 1 from `adminclient/dial.go::tokenSource`) which:
   - Caches the access token + expiry
   - On near-expiry (margin = 30s, matches existing CLI value), re-exchanges the cached assertion for a fresh token
   - On `codes.Unauthenticated` / `codes.PermissionDenied` from any RPC, invalidates the cached token and re-exchanges, then retries the call once
   - Source of assertions: clawkerd holds the CLI-signed assertion in memory after reading it from tmpfs at boot. **The assertion is NOT deleted from tmpfs at Register success** (only the verifier is — verifier is single-use, assertion is reusable until its TTL expires). On clawkerd boot the assertion is read into memory; tmpfs entry can be deleted then if we want a tighter window, but the CLI must mint with TTL long enough to cover normal container lifetime.
   - **Assertion TTL**: CLI mints with TTL = 24h (covers typical session lengths). Hydra accepts assertions up to their own configured cap. If a container outlives 24h, the next refresh attempt fails — clawkerd logs and exits; user re-runs `clawker container start` which mints a fresh assertion via re-AnnounceAgent. Document this expectation in `cmd/clawkerd/CLAUDE.md`.
   - Even though B4 only calls Register today, the wrapped client is correct from day one. Future RPCs (B5+) add zero plumbing — they just call methods on the same client.

4. **Graceful shutdown**: handle SIGTERM via signal.NotifyContext. `<-ctx.Done()` returns; clawkerd exits 0. No deregister RPC — registry evicts on Docker die via dockerevents. clawkerd shutdown is implied by container die from CP's POV.
