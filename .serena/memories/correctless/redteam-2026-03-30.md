## Critical Architectural Finding

**The firewall's root bypass creates a two-step attack path.** After gosu drops to claude, CapPrm/CapEff are zero — NET_ADMIN is unreachable. However, the UID 0 RETURN rule in iptables means any escalation to root immediately bypasses the firewall. Since `no-new-privileges` is not set, SUID escalation is a theoretical vector (though no exploitable SUID binary currently exists).

**Fix:** Add `no-new-privileges` as default security option. Drop NET_ADMIN from bounding set after firewall setup. Long-term: sidecar model.

## Top Fixes (Priority Order)

1. **Sidecar firewall model** — Remove NET_ADMIN/NET_RAW from agent container
2. **`no-new-privileges` securityOpt** — Prevent SUID escalation to root (which bypasses UID-scoped firewall)
3. **Package name validation** — `^[a-z0-9][a-z0-9.+\-]*$` regex reject
4. **Inject instruction allow-list** — Block RUN/COPY/ADD in inject blocks
5. **Bind mount `nosymfollow`** — Prevent symlink escape (Linux 5.10+)
6. **from_env allow-list** — Forward only explicitly safe env var names; don't rely on deny-lists
7. **Staging dir 0o700** — Fix world-readable credential temp dirs
8. **ip6tables fail-closed** — Remove `|| true` from IPv6 rules
