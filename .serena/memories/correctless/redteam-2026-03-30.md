## Critical Architectural Finding

**"The prisoner has the key"** — Clawker's firewall requires NET_ADMIN inside the agent container (for `firewall.sh enable` to set iptables rules). But NET_ADMIN also allows the container to flush those rules. The enforcement mechanism is co-located with the enforcement target.

**Fix:** Move firewall setup to a sidecar container or host-side nsenter. Don't give the agent container NET_ADMIN.

## Top Fixes (Priority Order)

1. **Sidecar firewall model** — Remove NET_ADMIN/NET_RAW from agent container
2. **`no-new-privileges` securityOpt** — Prevent SUID escalation to root (which bypasses UID-scoped firewall)
3. **Package name validation** — `^[a-z0-9][a-z0-9.+\-]*$` regex reject
4. **Inject instruction allow-list** — Block RUN/COPY/ADD in inject blocks
5. **Bind mount `nosymfollow`** — Prevent symlink escape (Linux 5.10+)
6. **from_env deny-list** — Block AWS_*, *_TOKEN, *_SECRET forwarding
7. **Staging dir 0o700** — Fix world-readable credential temp dirs
8. **ip6tables fail-closed** — Remove `|| true` from IPv6 rules
