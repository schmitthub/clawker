This guide takes you from zero to a fully working containerized Claude Code setup. By the end you'll have an isolated agent running in a Docker container with your credentials forwarded, a network firewall active, and your project code mounted.

## Prerequisites

- **Docker** must be installed and running
- An **Anthropic API key** or Claude Pro/Team subscription
- macOS or Linux (Windows is not currently supported)

## 1. Install Clawker

<CodeGroup>
```bash Homebrew
brew install schmitthub/tap/clawker
```

```bash curl
curl -fsSL https://raw.githubusercontent.com/schmitthub/clawker/main/scripts/install.sh | bash
```
</CodeGroup>

Verify it's installed:

```bash
clawker version
```

## 2. Initialize Your Project

```bash
cd your-project
clawker init
```

A guided setup walks you through:
1. **Project name** — defaults to the current directory name (must be lowercase)
2. **Language preset** — Python, Go, Rust, TypeScript, Java, Ruby, C/C++, C#/.NET, Bare, or "Build from scratch"
3. **Save or customize** — accept the preset as-is, or step through each field to tailor it

This creates:
- A `.clawker.yaml` config file pre-filled with the right base image, packages, and firewall domains for your language
- A `.clawkerignore` file (like `.dockerignore` for snapshot mode)
- A project entry in Clawker's registry so it always knows the project root

User settings (`~/.config/clawker/settings.yaml`) and XDG directories are bootstrapped automatically on first run.

<Tip>
To skip prompts and use the Bare preset with defaults, run `clawker init --yes`. You can always customize later with `clawker project edit`.
</Tip>

<Tip>
If you run `clawker init` inside a subdirectory of an existing project, it creates a layer config file that overrides the root config for that directory — useful for monorepos with per-service settings. See [Monorepo Support](/configuration#monorepo-support).
</Tip>

<Tip>
Language presets include ecosystem-specific firewall rules (e.g., `pypi.org` for Python, `registry.npmjs.org` for TypeScript). If you need additional domains, add them to `security.firewall.add_domains` or use `clawker firewall add` at runtime. See the [Firewall guide](/firewall#common-setups) for examples.
</Tip>

## 3. Build the Image

```bash
clawker build
```

This builds a Docker image tagged `clawker-<project>:latest` using the settings in your `.clawker.yaml` — base image, system packages, Claude Code, and any custom instructions. The image is content-addressed, so rebuilds only happen when the config changes.

## 4. Run Your First Agent

An **agent** is a named container instance running Claude Code. The `--agent` flag gives it a name for management.

```bash
clawker run -it --agent dev @
```

- `-it` — interactive mode with a terminal attached
- `--agent dev` — names this container `clawker.<project>.dev`
- `@` — shortcut that resolves to your project's built image

You're now inside a containerized Claude Code session. Your project code is bind-mounted (live sync), your Git credentials are forwarded, and the network firewall is active.

### What Happens Behind the Scenes

When you run this command, Clawker:

1. Creates the container with your project directory mounted at its real host path
2. Creates persistent volumes for Claude Code config (`~/.claude`) and shell history
3. Starts the firewall daemon (Envoy + CoreDNS) if it isn't already running
4. Applies iptables rules inside the container to redirect traffic through the firewall
5. Copies your host Claude Code settings, plugins, and authentication into the container
6. Forwards SSH/GPG agent sockets and Git HTTPS credentials from your host
7. Runs any `post_init` script you've configured
8. Launches Claude Code

The agent prompt at `/etc/claude-code/CLAUDE.md` is automatically loaded, so the agent knows it's in a container and can help troubleshoot firewall blocks.

## 5. Working With Your Agent

### Detach Without Stopping

Press `Ctrl+P`, then `Ctrl+Q` to detach. The container keeps running in the background.

### Re-attach

```bash
clawker attach --agent dev
```

### Stop the Agent

Exit Claude Code with `/exit`, or from another terminal:

```bash
clawker stop --agent dev
```

### Restart a Stopped Agent

```bash
clawker start -a -i --agent dev
```

The `-a` (attach) and `-i` (interactive) flags reconnect your terminal. Your session data, config, and command history are preserved across restarts because they live in persistent volumes.

### One-off Disposable Agent

Use `--rm` to automatically remove the container when it stops — good for quick tasks:

```bash
clawker run -it --rm --agent scratch @ --dangerously-skip-permissions
```

Arguments after the image (`@`) are passed directly to Claude Code inside the container.

## 6. Customize Your Config

You can edit `.clawker.yaml` directly, or use the interactive TUI editor:

```bash
clawker project edit
```

Here's a typical setup for a Node.js project:

```yaml
build:
  image: "node:20-slim"
  packages:
    - git
    - curl
    - ripgrep

agent:
  env:
    NODE_ENV: "development"
  post_init: |
    npm install -g typescript

security:
  firewall:
    add_domains:
      - github.com
      - api.github.com
      - registry.npmjs.org
    rules:
      - dst: github.com
        proto: ssh
        port: 22
        action: allow
```

After changing the config, rebuild:

```bash
clawker build
```

See the [Configuration](/configuration) guide for the full reference.

<Tip>
**Claude Code users:** Install the clawker-support plugin for hands-on help with setup, configuration, MCP wiring, firewall rules, and troubleshooting. It acts as a clawker internals expert — researches what you're trying to add, reads the real Dockerfile template and config schema, and gives you the exact YAML you need.

```bash
claude plugin marketplace add schmitthub/claude-plugins
claude plugin install clawker-support@schmitthub-plugins
```
</Tip>

## 7. Parallel Agents with Worktrees

Run multiple agents on separate branches using Git worktrees:

```bash
# Agent on a feature branch (creates worktree automatically)
clawker run -it --agent auth --worktree feature/auth:main @ --dangerously-skip-permissions

# Another agent on a different branch
clawker run -it --agent cache --worktree feature/cache:main @ --dangerously-skip-permissions
```

The `--worktree branch:base` flag creates a Git worktree off the base branch and mounts it into the container. Each agent works on its own branch with no conflicts.

```bash
# See all worktrees
clawker worktree list

# Clean up when done
clawker worktree remove --delete-branch feature/auth
```

See the [Worktrees](/worktrees) guide for more.

## 8. Monitoring (Optional)

Start the monitoring stack before your agents to get real-time dashboards:

```bash
clawker monitor init    # Scaffold configs (one-time)
clawker monitor up      # Start Grafana, Prometheus, Loki, Jaeger
```

Open Grafana at `http://localhost:3000` to see costs, token usage, tool calls, and session timelines for every agent.

```bash
clawker monitor down    # Stop when done
```

See the [Monitoring](/monitoring) guide for details.

## 9. Autonomous Loops (Experimental)

Run Claude Code in iterative loops with a fresh container per iteration:

```bash
clawker loop iterate \
  --prompt "Fix all failing tests and open a PR" \
  --max-loops 10 \
  --skip-permissions \
  --worktree fix/tests:main
```

A TUI dashboard shows progress, tool calls, costs, and circuit breaker status in real time. See the [Loop Mode](/loops) guide for configuration.

## Managing Resources

Clawker mirrors Docker CLI patterns but only operates on Clawker-managed resources:

```bash
clawker ps                            # List containers
clawker image ls                      # List images
clawker volume ls                     # List volumes

clawker stop --agent dev              # Stop a container
clawker rm --agent dev                # Remove a container
clawker volume prune                  # Clean up orphaned volumes
```

### Firewall Management

```bash
clawker firewall status               # Health check
clawker firewall list                 # Show active rules
clawker firewall add pypi.org         # Allow a domain at runtime
clawker firewall remove pypi.org      # Revoke access
clawker firewall bypass 5m --agent dev  # Temporary unrestricted access
```
